02 — Datasets

What Data We Have

Five JSON files form the complete intelligence knowledge base. Every file cross-references the others by ID — together they form a traversable graph.

📁 datasets/threat_entities.json
The entity registry — 21 objects across 5 types. Every actor, malware sample, infrastructure node, vulnerability, and target sector the platform knows about. IDs referenced by all other files.
7 threat actors 8 malware families 2 infrastructure nodes 2 vulnerabilities 3 target sectors risk_score 0–1 STIX-compatible IDs
📁 datasets/attack_patterns.json
15 OSINT-specific attack patterns with MITRE ATT&CK technique references, kill chain phase, detection guidance, and mitigation. Covers everything from metadata exposure to dark web monitoring.
15 patterns OSINT-T001 → T015 MITRE technique refs kill_chain_phase detection + mitigation risk_level 0–1
📁 datasets/relations.json
28 directed edges that turn flat entity lists into a graph. Maps: actor → uses → malware, actor → exploits → CVE, actor → targets → sector, actor → uses_pattern → attack pattern. This is what graph_builder.py reads to build the node graph.
28 edges 4 relation types confidence score first/last_observed source references
📁 datasets/mitre_mapping.json
7 actor-level ATT&CK profiles. Each maps an actor to their specific techniques with frequency ratings (very_high / high / medium), tactic, kill chain phase, and analyst notes. Bridges threat actors to the ATT&CK framework for TTP-based detection.
7 actor profiles 40+ techniques frequency ratings kill_chain_phase mitre_group_id osint_patterns_used
📁 datasets/campaigns.json
7 real-world campaigns structured using the Diamond Model of Intrusion Analysis. Each connects adversary + capability + infrastructure + victim with timeline, kill chain coverage, impact assessment, and attribution confidence. The highest-level view of how actors operate.
7 campaigns Diamond Model kill chain checklist financial impact attribution confidence vulnerability refs
core_engine/graph_builder.py — How the three base datasets become a graph 
import json, networkx as nx
from pathlib import Path

class GraphBuilder:
    """Reads the 3 base datasets and builds a NetworkX property graph."""

    def __init__(self, datasets_path: str = "datasets/"):
        self.path = Path(datasets_path)
        self.G = nx.DiGraph()   # Directed graph: actor → malware, actor → sector

    def build(self) -> nx.DiGraph:
        # 1. Load all 5 dataset files
        entities  = self._load("threat_entities.json")
        patterns  = self._load("attack_patterns.json")
        relations = self._load("relations.json")
        mitre     = self._load("mitre_mapping.json")
        campaigns = self._load("campaigns.json")

        # 2. Add entity nodes with all attributes as node properties
        for e in entities:
            self.G.add_node(e["entity_id"], **e)

        # 3. Add attack patterns as nodes
        for p in patterns:
            self.G.add_node(p["pattern_id"], **p)

        # 4. Add directed edges from relations.json
        for r in relations:
            self.G.add_edge(
                r["source_id"], r["target_id"],
                relation_type=r["relation_type"],
                confidence=r["confidence"]
            )

        # 5. Attach MITRE TTP data to existing actor nodes
        for m in mitre:
            if m["actor_id"] in self.G:
                self.G.nodes[m["actor_id"]]["mitre_techniques"] = m["techniques"]

        return self.G

    def _load(self, filename) -> list:
        with open(self.path / filename) as f:
            return json.load(f)
core_engine/risk_analyzer.py — How risk scores are calculated 
class RiskAnalyzer:
    """
    Risk Score = (Threat Likelihood × Impact) × Confidence Weight
    Outputs 0.0–1.0 mapped to: LOW | MEDIUM | HIGH | CRITICAL
    """

    SEVERITY = {
        (0.9, 1.0): "CRITICAL",
        (0.7, 0.9): "HIGH",
        (0.4, 0.7): "MEDIUM",
        (0.0, 0.4): "LOW"
    }

    def score_entity(self, entity: dict, graph) -> dict:
        # Likelihood: how often has this actor targeted this sector?
        likelihood = entity.get("risk_score", 0.5)

        # Impact: CIA triad — Confidentiality, Integrity, Availability
        impact = self._calc_impact(entity)

        # Confidence: how reliable is the intelligence source?
        conf = entity.get("confidence", 0.5)

        score = min(likelihood * impact * conf, 1.0)
        severity = self._band(score)

        return {
            "entity_id": entity["entity_id"],
            "risk_score": round(score, 3),
            "severity": severity,
            "rationale": self._explain(entity, score, graph)
        }

    def _explain(self, entity, score, graph) -> str:
        # explainability.py uses this to generate human-readable rationale
        neighbors = list(graph.successors(entity["entity_id"]))
        return (
            f"Score {score:.2f} based on: risk_score={entity.get('risk_score')}, "
            f"confidence={entity.get('confidence')}, "
            f"connected_nodes={len(neighbors)}"
        )
