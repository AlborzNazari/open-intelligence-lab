// 02 ‚Äî Bottom-Up

From Bytes to Intelligence

How raw network data becomes actionable intelligence. Each step transforms, enriches, and correlates signals.

1. Raw Data Primitives ‚Äî IP, Domain, Username, Email, Hash
The atomic input types the system accepts. These are validated via Pydantic schemas before any processing begins. Malformed inputs are rejected at the boundary, not deep in the stack.
2. Adapter Layer ‚Äî Normalized API Calls to External Sources
Each connector wraps a third-party API or scraper. Responses are normalized into a shared IntelResult schema. Rate-limiting, retries, and auth are handled here ‚Äî never in business logic.
3. Module Layer ‚Äî Domain-Specific Intelligence Extraction
Modules compose multiple adapter calls. A DomainIntel module might call WHOIS + DNS + VirusTotal + Shodan and merge results. This is where correlation and deduplication happen.
4. Orchestration ‚Äî Pipeline Assembly & Execution
The dispatcher decides which modules run, in what order, with what dependencies. Supports sequential, parallel, and conditional execution paths depending on initial query type.
5. Presentation ‚Äî Structured JSON / Graph / CLI Output
Results are serialized into clean JSON responses or rendered as terminal tables. The entity relationship graph can be exported for visualization in tools like Gephi or D3.js.

from pydantic import BaseModel
from typing import Optional, List, Dict, Any
from datetime import datetime

class IntelResult(BaseModel):
    # Every module output conforms to this contract
    query: str                          # Original input (IP, domain, email)
    query_type: str                     # "ip" | "domain" | "email" | "username"
    source: str                         # Which adapter produced this
    confidence: float                   # 0.0‚Äì1.0 reliability score
    timestamp: datetime
    data: Dict[str, Any]               # Source-specific payload
    related_entities: List[str] = []   # For graph building
    tags: List[str] = []              # e.g. ["malicious","vpn","tor"]
    error: Optional[str] = None

class IntelReport(BaseModel):
    target: str
    results: List[IntelResult]
    entity_graph: Dict[str, List[str]]  # Adjacency list
    risk_score: float
    generated_at: datetime
modules/domain_intel.py ‚Äî A full intelligence module 
import asyncio
from adapters import whois_adapter, dns_adapter, shodan_adapter, virustotal_adapter
from models import IntelResult, IntelReport

class DomainIntelModule:
    """
    Aggregates multi-source intelligence for a domain.
    Uses async to fan-out parallel API calls, then merges results.
    """

    async def run(self, domain: str) -> IntelReport:
        # Fan-out: all adapters run concurrently
        results = await asyncio.gather(
            whois_adapter.lookup(domain),
            dns_adapter.resolve(domain, record_types=["A","MX","TXT","NS"]),
            shodan_adapter.search_domain(domain),
            virustotal_adapter.check_domain(domain),
            return_exceptions=True
        )

        # Filter failed adapters, don't let one failure kill the report
        valid = [r for r in results if isinstance(r, IntelResult)]

        # Build entity graph from related entities across all results
        graph = self._build_graph(domain, valid)
        risk = self._calculate_risk(valid)

        return IntelReport(
            target=domain, results=valid,
            entity_graph=graph, risk_score=risk
        )

    def _calculate_risk(self, results) -> float:
        # Weighted scoring: VT detections carry highest weight
        weights = {"virustotal": 0.5, "shodan": 0.3, "whois": 0.2}
        score = sum(r.confidence * weights.get(r.source, 0.1) for r in results)
        return min(score, 1.0)
api/routes.py ‚Äî FastAPI layer (thin, delegates everything) 
from fastapi import FastAPI, HTTPException
from modules.domain_intel import DomainIntelModule
from modules.ip_intel import IPIntelModule

app = FastAPI(title="Open Intelligence Lab API", version="1.0.0")

@app.get("/intel/domain/{domain}")
async def domain_intel(domain: str):
    try:
        module = DomainIntelModule()
        report = await module.run(domain)
        return report.dict()
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

@app.get("/intel/ip/{ip}")
async def ip_intel(ip: str):
    report = await IPIntelModule().run(ip)
    return report.dict()

// 09 ‚Äî Engineering Tradeoffs

What a Senior Engineer
Would Ask You

QuestionStrong Answer
Why Python and not Go/Rust? Python has the richest OSINT ecosystem ‚Äî requests, beautifulsoup, shodan SDK, pydantic. For I/O-bound work, asyncio gives us parallelism without the complexity of compiled languages. If throughput becomes a bottleneck, a Go gateway can front the Python workers.
How do you handle rate limiting? Each adapter implements its own retry-with-backoff strategy using tenacity. API keys rotate via environment variables. Rate limits are configurable per adapter so aggressive queries can be tuned down in production.
What's your data freshness strategy? Currently no caching ‚Äî every query hits live APIs. The next step is Redis-backed TTL caching at the adapter level. WHOIS data can be cached for 24h; VirusTotal results for 1h. This would cut API costs by ~60%.
How do you test this? Unit tests mock adapter responses with known fixtures. Integration tests run against a sandbox API environment. The Pydantic models serve as a contract test ‚Äî if an adapter returns unexpected structure, it fails at parse time, not at runtime logic.
Can it be weaponized? Yes ‚Äî any OSINT tool can. Mitigations: rate-limiting at the API layer, no anonymous access in production, logging all queries for audit, and scoping the tool to domains/IPs rather than enabling bulk crawling. A responsible use policy is the right first line of defense.
